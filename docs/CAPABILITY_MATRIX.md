# V1.3.2 Capability Matrix

## Supported

| Capability | Status | Notes |
|---|---:|---|
| SML parsing | Supported | Parses project, stack, models, routes, policies, constraints, and generalized V1.3 sections. |
| Semantic validation | Supported | Validates structure before code generation and emits proof/fact traces. |
| Deterministic FastAPI generation | Supported | Final source code is generated by the compiler, not directly by an LLM. |
| AppFoundationPlan | Supported | Converts app prompts into structured app-foundation plans through deterministic/offline logic and online-compatible contracts. |
| Domain-pack selection | Supported | Selects packs such as generic CRUD, HR, expense, CRM, ticketing, school, procurement, and inventory benchmarks. |
| Capability negotiation | Supported | Returns supported/partial/unsupported diagnostics before generation. |
| Full CRUD route generation | Supported | GET list, POST create, GET by id, PATCH/PUT update, and DELETE are generated with typed service/repository/route layers. |
| Basic relationships | Supported | Basic relationship declarations are recognized and carried through foundation negotiation and generated docs. |
| Multi-domain benchmarks | Supported | Inventory, Todo, HR Leave, Expense Approval, CRM Pipeline, Ticketing, and School Records examples validate and compile. |
| Generated quality gates | Supported | Tests, coverage, Ruff, format, mypy, compileall, Bandit, pip-audit, and Alembic cycle are part of release validation. |
| Online-build mock loop | Supported | Mock online provider path can draft, negotiate, compile, run quality gates, and return ACCEPTED. |
| Boundary-aware secret scan | Supported | Scans generated artifacts and project files while avoiding false positives on ordinary slugs. |

## Partially supported

| Capability | Status | Notes |
|---|---:|---|
| Live DeepSeek/Gemini/OpenAI builds | Partial | Provider layer exists; default release E2E skips live calls unless `RUN_DEEPSEEK_LIVE=1` and credentials are supplied. |
| PostgreSQL integration | Partial | Docker/PostgreSQL path exists and is gated by `RUN_POSTGRES=1`; default local run uses SQLite migration cycle. |
| Workflow semantics | Partial | SML sections and benchmark semantics exist; full runtime workflow/state-machine executor is V1.4 scope. |
| Tenant/RBAC primitives | Partial | Planning/SML primitives exist; full generated tenant-scoped runtime enforcement is V1.4 scope. |
| Audit primitives | Partial | Audit/logging structures exist; full product-grade audit event pipeline is V1.4 scope. |

## Out of scope for V1.3.2

| Capability | Status | Notes |
|---|---:|---|
| Frontend/UI generation | Out of scope | Candidate V1.5 scope. |
| Hosted SaaS control plane | Out of scope | Candidate commercial/product layer. |
| Arbitrary third-party integrations | Out of scope | Should be modeled through future integration packs. |
| Full no-code product | Out of scope | V1.3.2 is a backend foundation compiler, not a full no-code platform. |
| Payment/tax/accounting engines | Out of scope | Requires separate regulated-domain packs and stronger invariants. |

## Release claim

V1.3.2 can be described as:

```text
A general-purpose semantic app foundation compiler that turns app ideas into validated SML and deterministic FastAPI backend foundations with quality-gated generated output.
```
